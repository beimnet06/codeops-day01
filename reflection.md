# Reflection Questions

## 1. What is the difference between Git and GitHub?
Git is a version control system that runs locally on your computer to track changes in files. GitHub is a cloud-based hosting service that stores Git repositories online and provides collaboration features.

## 2. What is the purpose of the terminal?
The terminal is a text-based interface that allows users to interact with the computer by typing commands. It provides direct control over the operating system and enables automation of tasks.

## 3. What does git add do?
git add stages changes (new files, modifications, or deletions) to be included in the next commit. It tells Git which changes you want to save in the repository.

## 4. What is the purpose of git commit?
git commit creates a permanent snapshot of the staged changes with a unique ID and timestamp. It saves the current state of your project with a descriptive message explaining what was changed.

## 5. Write the daily Git workflow in the correct order.
1. git status - Check current state of repository
2. git add . - Stage all changes
3. git commit -m "message" - Commit changes with a message
4. git push - Push commits to remote repository
